CLAUDE.md

Project Instructions

Claude Code가 프로젝트에 들어왔을 때 자동으로 읽는 지시 파일이다. README처럼 사람이 읽는 문서가 아니라, Claude에게 "이 프로젝트에서 어떻게 행동해야 하는가"를 알려주는 AI용 컨텍스트 주입 파일이다. 모든 대화 세션에서 자동 로드되므로 매번 배경을 설명하지 않아도 된다.

개념

  • 프로젝트 루트에 위치 — 자동 로드
  • git에 커밋되어 팀 전체에 공유 가능
  • Claude의 행동 규칙, 금기사항 정의
  • 아키텍처, 실행 명령, 컨벤션 문서화
  • 중첩 디렉터리에도 배치 가능 (범위 제한)

이 프로젝트의 활용

  • 목적 설명: 부산대 생성모델 수업 과제 → 연구 도구
  • 아키텍처 맵: fetcher → summarizer → app 흐름
  • 컨벤션 강제: GEMINI_API_KEY를 코드에 넣지 말 것
  • 모델 관리: config.yaml에서만 모델 변경
  • API 예절: delay_seconds=3 이하로 줄이지 말 것
## 컨벤션 (CLAUDE.md 일부) - GEMINI_API_KEY는 절대 코드/git에 들어가지 않는다. - 모델 선택은 config.yaml의 summary_model에서만 관리. - arXiv API: delay_seconds=3, 더 짧게 줄이지 말 것. - 새 SQL 컬럼 → SCHEMA 수정 + 마이그레이션 함수 같이 작성.

Custom Slash Commands

Automation

.claude/commands/ 디렉터리에 마크다운 파일을 두면 Claude Code에서 /파일명으로 바로 호출할 수 있다. 반복적인 작업 흐름을 한 번의 명령으로 자동화하는 일종의 매크로 + 프롬프트 템플릿이다. frontmatter로 허용 도구, 인자 힌트 등을 설정할 수 있다.

사용자: /daily_papers
→
src/fetcher 실행
→
src/summarizer 실행
→
DB에서 상위 5편 출력

/daily_papers

  • fetch + summarize 한 번에 실행
  • 최신 논문 5편 한국어 요약 출력
  • 허용 도구: Bash, Read
  • GEMINI_API_KEY 없으면 자동 안내
  • 매일 아침 루틴으로 설계됨

/search_topic <키워드>

  • 일회성 arXiv 즉석 검색 (DB 저장 안 함)
  • cs.CL / cs.LG / cs.AI 카테고리 제한
  • 인자 없으면 키워드 입력 유도
  • 허용 도구: Bash, Read, Write
  • paper-summary 형식으로 요약 통일

/my_save — 세션 진행상황 저장

  • 이번 세션에서 변경된 파일, 결정, 통찰 검토
  • 반복 적용될 규칙만 CLAUDE.md에 추가 (일회성 내용 제외)
  • 기존 섹션 구조를 유지하며 적합한 위치에 삽입
  • 추가할 내용이 없으면 수정하지 않고 사유를 사용자에게 보고
--- (search_topic.md frontmatter) description: 특정 주제 키워드로 arXiv를 즉석에서 검색 allowed-tools: Bash, Read, Write argument-hint: <검색할 주제 키워드> --- 사용자가 인자로 넣은 주제 $ARGUMENTS 에 대해 다음을 수행하라: 1. arxiv 패키지로 최신 논문 5편 수집 (cs.CL, cs.LG, cs.AI) 2. paper-summary 스킬 형식으로 한국어 요약 3. 결과 출력 (DB에는 저장하지 않음)

Skills

Reusable Behavior

Slash Command가 "실행 흐름의 자동화"라면, Skill은 Claude의 출력 형식·행동 방식을 표준화하는 재사용 가능한 지시 템플릿이다. .claude/skills/<이름>/SKILL.md에 위치하며, 여러 Command에서 공유해서 호출할 수 있다. "이 작업은 항상 이 방식으로 해야 한다"는 규칙을 캡슐화한다.

개념

  • 출력 포맷·규칙을 한 곳에서 관리
  • 여러 Command에서 동일 품질 보장
  • 자동 트리거 조건 설정 가능
  • Command보다 세밀한 행동 지침

paper-summary 스킬 적용 대상

  • /daily_papers — 일일 논문 요약
  • /search_topic — 일회성 검색 요약
  • paper-evaluator agent — 평가 전 요약
  • 사용자가 "요약해줘" 요청 시 자동 적용
## paper-summary 출력 형식 (항상 이 구조를 따름) ## 핵심 기여 이 논문이 새롭게 제안하거나 보여주는 것. 1~3문장. ## 방법론 어떤 모델/기법/데이터셋을 사용했는지. 수식 피하고 핵심 아이디어. ## 결과 실험 결과. 수치는 그대로 인용. ## 한계 명시되거나 추정되는 한계. 1~2문장. ## 키워드 관련 키워드 3~5개 (쉼표로 구분)

작성 규칙 5가지:

추측 금지 1~3문장 간결성 모델명 영어 보존 수치 그대로 인용 자연스러운 한국어

Subagents

Specialized AI

특정 역할에 특화된 별도 Claude 인스턴스다. 메인 Claude와 독립된 컨텍스트로 동작하며, 정해진 역할·지침·모델·허용 도구를 가진다. .claude/agents/<이름>.md에 정의하고 메인 Claude가 필요 시 자동으로 호출한다. 복잡한 판단을 전문 에이전트에 위임해 메인 컨텍스트를 아끼는 설계 패턴이다.

paper-evaluator 에이전트

  • 모델: claude-sonnet
  • 허용 도구: Read, Bash (DB 접근용)
  • 트리거: "점수 매겨줘", "읽을 가치 있어?"
  • arxiv_id 또는 abstract 텍스트 입력 허용

평가 기준 (가중 평균)

  • 관심 적합도 40% — config.yaml 키워드 매칭
  • 신규성 30% — 기존 방법과의 차별점
  • 실용성 20% — 실제 적용 가능성
  • 임팩트 10% — 후속 연구 영향력
## 출력 예시 점수: 8/10 사유: RAG 파이프라인 개선에 직접 적용 가능한 방법론을 제시하며, config.yaml의 핵심 키워드 "retrieval-augmented"와 정확히 일치. 읽기 추천: 예

Command와 Agent의 차이: Command는 "무엇을 실행할지" 정의, Agent는 "누가 어떻게 판단할지" 정의. daily_papers 명령이 끝난 후 각 논문을 paper-evaluator가 채점해서 우선순위를 정하는 흐름으로 연계된다.

Hooks

Event Triggers

Claude Code의 특정 이벤트(생명주기 시점)에 자동으로 실행되는 쉘 커맨드 또는 프롬프트다. .claude/settings.json의 hooks 키에 설정하며, "Claude가 응답을 마쳤을 때", "컨텍스트가 압축되기 직전" 같은 이벤트에 반응한다. 사용자가 직접 요청하지 않아도 자동으로 실행된다는 점에서 CI/CD 훅과 유사하다.

PreCompact 훅 — 자동 백업

  • 트리거: 대화 컨텍스트가 압축되기 직전
  • 동작: 세션 중요 내용을 CLAUDE.md에 기록
  • 목적: 컨텍스트 소실 전 영구 저장
  • 기록 항목: 날짜(년/월/일/시간) 포함
  • 적용 범위: 이 프로젝트에만 (local)

Stop 훅 — 완료 알림

  • 트리거: Claude가 응답 완료할 때마다
  • 동작: 시스템 beep 음 출력
  • 목적: 긴 작업(summarizer 실행 등) 완료 알림
  • fetcher/summarizer 같은 장시간 작업에 유용
  • 적용 범위: 이 프로젝트에만 (local)
// .claude/settings.json { "hooks": { "PreCompact": [{ "type": "prompt", "prompt": "세션의 중요한 내용을 CLAUDE.md에 날짜와 함께 기록해줘" }], "Stop": [{ "type": "command", "command": "[console]::beep(800, 300)" }] } }

훅 설정은 /update-config 자연어 명령으로 손쉽게 추가할 수 있다. 세부 설정 확인은 cat .claude/settings.json으로 한다.

Auto Memory

Persistent Context

대화 세션이 끝나도 사라지지 않는 파일 기반 장기 기억 시스템이다. Claude가 자동으로 사용자 정보, 프로젝트 맥락, 피드백, 외부 리소스 링크를 ~/.claude/projects/<프로젝트>/memory/에 저장하고, 다음 세션에서 로드한다. "저번에 했던 그거…"를 매번 재설명하지 않아도 되는 핵심 기능이다.

👤
user
사용자 역할, 경험, 선호도
예: "부산대 생성모델 수강생"
💬
feedback
협업 방식 교정 사항
예: "이 방식으로 해달라"
🗂️
project
진행 중 작업, 결정 이유
예: "수업 과제 → 연구 도구"
## MEMORY.md (인덱스 파일 — 200줄 제한) - [user_profile.md](user_profile.md) — 사용자 정보 및 환경 설정 - [project_fixes.md](project_fixes.md) — 초기 실행 버그 수정 내역 --- ## user_profile.md (개별 기억 파일 예시) --- name: user-profile type: user --- 부산대 생성모델 수업 수강생. Python 3.11 사용. Why: arXiv 논문 추적기를 과제 + 연구 도구로 제작 How to apply: LLM 관련 설명 시 연구 맥락에서 설명

CLAUDE.md vs Memory의 차이:

구분 CLAUDE.md Auto Memory
위치 프로젝트 루트 (git 관리) ~/.claude/projects/ (로컬 전용)
내용 프로젝트 규칙, 아키텍처, 컨벤션 사용자 프로필, 피드백, 세션 맥락
수정 주체 사람 + Claude (/my_save로) Claude가 자동으로 판단 후 저장
공유 여부 팀 전체 공유 가능 나만 사용 (개인화)
전체 흐름 — 기술들이 연결되는 방식

통합 아키텍처

How It Fits Together

6가지 기술은 독립적으로 쓰이지 않고 서로 연계되어 동작한다.

📄 CLAUDE.md 로드
→ Claude에 프로젝트 규칙 주입 (세션 시작 시 자동)
🧠 Memory 로드
→ 사용자 프로필, 이전 세션 맥락 복원
⚡ /daily_papers
→ fetcher 실행 → summarizer 실행 → 결과 표시
🔧 paper-summary 적용
→ 5섹션 형식 강제 (요약 품질 일관성 보장)
🤖 paper-evaluator
→ 각 논문 1~10점 채점 (읽기 우선순위 결정)
🪝 Stop 훅 실행
→ 완료 beep 알림 (긴 작업 종료 감지)
⚡ /my_save
→ 세션 결정사항을 CLAUDE.md에 영구 반영
기술역할파일 위치호출 방식
📄 CLAUDE.md 프로젝트 규칙·컨벤션 전달 프로젝트 루트 자동 로드
⚡ Slash Commands 반복 작업 자동화 .claude/commands/ /명령어 입력
🔧 Skills 출력 형식 표준화 .claude/skills/ 자동 트리거 or 명시 호출
🤖 Subagents 전문 판단 위임 .claude/agents/ 메인 Claude가 자동 호출
🪝 Hooks 이벤트 기반 자동 실행 .claude/settings.json 이벤트 발생 시 자동
