# paper-summary

논문을 한국어로 요약할 때는 **항상** 아래 형식을 따를 것.

## 출력 형식 (이 헤더 그대로 사용)

```
## 핵심 기여
이 논문이 새롭게 제안하거나 보여주는 것. 1~3문장.

## 방법론
어떤 모델/기법/데이터셋을 사용했는지. 1~3문장. 수식은 피하고 핵심 아이디어 위주.

## 결과
어떤 실험 결과를 얻었는지. 가능하면 수치를 그대로 인용. 1~3문장.

## 한계
명시되거나 합리적으로 추정되는 한계점. 1~2문장.

## 키워드
관련 키워드 3~5개 (쉼표로 구분)
```

## 작성 규칙

1. **추측 금지**: abstract에 명시되지 않은 내용은 "abstract만으로는 불명확"이라고 적는다. 본인이 아는 일반 지식으로 메꾸지 않는다.
2. **간결성**: 각 섹션은 1~3문장. 마케팅 문구("획기적인", "혁신적인" 등)는 쓰지 않는다.
3. **고유명사 보존**: 모델명/데이터셋명/벤치마크명은 영어 원문 그대로 (예: GPT-4, MMLU, GSM8K).
4. **숫자 보존**: abstract에 나온 수치(정확도, 파라미터 수 등)는 그대로 인용한다.
5. **한국어 자연스러움**: 직역체("~을 수행한다") 대신 자연스러운 한국어("~한다", "~했다").

## 예시

입력: "We introduce GPT-4, a large multimodal model... achieves human-level performance on various professional benchmarks..."

출력:
```
## 핵심 기여
이미지와 텍스트를 모두 입력받는 대규모 멀티모달 모델 GPT-4를 제안하고, 다양한 전문 벤치마크에서 인간 수준의 성능을 달성했다.

## 방법론
abstract만으로는 불명확하나 대규모 transformer 기반 모델로 추정. 텍스트와 이미지를 함께 입력으로 받는다.

## 결과
여러 전문 벤치마크에서 인간 수준 성능을 보고. 구체 수치는 abstract만으로는 불명확.

## 한계
abstract에는 명시되지 않음. 일반적으로 대규모 모델의 학습 비용과 환각 문제가 한계로 거론됨.

## 키워드
GPT-4, multimodal, benchmark, large language model, human-level performance
```
